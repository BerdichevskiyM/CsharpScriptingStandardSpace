# ScriptingStandardSpace

*Немного терминов:*
1) Функции - функции в классе или структуре или вне его, которые возвращают значение.
2) Методы - функции в классе или структуре или вне его, которые не возвращают значение.
3) Дельта - параметр в функциях, который используется в качестве уменьшения состояния. Может принимать любой тип, имя параметра - «delta», имя функции может содержать выражение «ByDelta». Также существует как поле или свойство.
4) Зета - параметр в функциях, который используется в качестве увеличения состояния. Может принимать любой тип, имя параметра - «zeta», имя функции может содержать выражение «ByZeta». Также существует как поле или свойство.
5) Значение - параметр в функциях, который используется в качестве присваивания состояния. Может принимать любой тип, имя параметра - «value», имя функции может содержать выражение «ByValue». Также существует как поле или свойство.

*Объяснение:*

Скрипт может описывать только один класс, структуру или интерфейс, бесконечное число других типов (перечисления, подклассы и т.д.), которые связаны с данным классом, структурой или интерфейсом.

В названии объектов не допускаются ключевые слова, которые принадлежат их типам («Class», «Struct» и т.д.), и слово «Script».

Пример оформления скрипта начинать смотреть с «Program.cs», а потом уже «Fabs.cs».

Поля, свойства, методы, функции и т.д. в первую очередь упорядочиваются по типу значения (у функций это возвращаемое значение) в порядке убывания зависимостей. Т.е. первым идёт тип, созданный пользователем; вторым - тип из плагина; третим - тип из движка (фреймворка); четвёртым - тип C#.

Вторая сортировка - сначала ссылочный тип, потом тип значения.

Третья сортировка - порядок абстрактности: abstract → virtual → override.

И наконец чётвертая сортировка - порядок модификаторов доступа по убыванию: public → protected internal → internal → protected → private protected → private. Последний употребляется только в интерфейсах.
